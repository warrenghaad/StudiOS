# STUDiOS

STUDiOS is the main trunk for organizing learning apps, support repos, curriculum work, and future IDE workflows. This repository is the coordination hub for the larger STUDiOS ecosystem, not the place where every app must live.

## What STUDiOS organizes

STUDiOS brings together projects for:

- drawing
- language
- music
- math
- mechanics
- curriculum design
- LMS development
- etextbook building
- image processing

## What this repository is for

Use this repository to:

- define the overall vision for STUDiOS
- track active repos and Replit projects
- separate platform work from content work
- document version progression across projects
- standardize how new projects enter the ecosystem

## Naming boundary

Inside this repository, use **STUDiOS** as the project and brand name.

The GitHub repository is currently named `StudiOS`. Renaming the GitHub repository to `STUDiOS` is a separate decision and can be made later if full naming consistency is important.

## Platform vs content

STUDiOS separates two kinds of work:

- **Platform**: repos, app infrastructure, interfaces, workflows, pipelines, and IDE setup
- **Content**: lessons, curriculum, learning pathways, etextbooks, and research-driven educational material

See `/docs/platform-and-content.md` for the working boundary.

## Current project inventory

## Working approach

1. Keep projects in their own repositories when that is simplest.
2. Use STUDiOS to describe, connect, and prioritize them.
3. Import, mirror, or automate projects only after the organization model is stable.
