# Platform and Content Boundary

STUDiOS needs a clear working split between the systems that deliver learning and the material that learners move through.

## Platform

Platform work includes:

- application repositories
- UI systems
- asset pipelines
- database and graph tooling
- workflow automation
- IDE and workspace setup
- version tracking across builds

Platform examples in the current ecosystem:

- STUDiOS
- DatabaseS-UI
- Content-Pipeline-And-reconciliation-Tool
- HTML-CSS-JS-Static-1
- WKSP-1-Ingest-Graph-Review

## Content

Content work includes:

- lesson ideas
- curriculum maps
- etextbook structures
- research-driven learning materials
- subject-specific pathways
- assembled educational artifacts

Content examples in the current ecosystem:

- geometric-civilizations
- Artifact-Research
- Artifact-Research-Timelinev1
- Artifact-Research-Timelinev3

## Hybrid projects

Some projects are both platform and content because they teach through interaction while also acting as software products.

Examples:

- ArtisanalCanvas
- MusicTutor
- Harmony-Tutor
- TripSpanishTutor
- Geometric-Math-Visualizer
- Inventive-Worlds

## Working rule

When a new project is added, decide first whether it is:

1. mainly platform
2. mainly content
3. hybrid

That decision should guide where it sits in the STUDiOS inventory and what success looks like for the project.
