# STUDiOS Project Intake

Use this process whenever a new repo, Replit build, or experiment enters the STUDiOS ecosystem.

## Intake fields

Record the following:

1. project name
2. category
3. learning goal
4. project type
5. repo or Replit link
6. current state
7. relationship to other STUDiOS projects
8. next step

## Category options

- drawing
- language
- music
- math
- mechanics
- curriculum
- LMS
- etextbooks
- image processing
- platform
- research

## Project types

- GitHub repository
- Replit project
- hybrid repo and Replit build
- research source
- archived legacy version

## Intake checklist

- decide whether the project is platform, content, or hybrid
- assign the main category
- add the project to `/docs/project-index.md`
- mark its current status
- name its next step in one sentence
- note whether it replaces, extends, or depends on another project
